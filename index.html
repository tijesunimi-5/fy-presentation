The Security Context

Nigeria's contemporary security landscape has undergone a distressing transformation, evolving from localized criminal activities into a complex crisis[cite: 45]. In recent years, life-threatening situations including kidnapping and violent assault have reached levels constituting a national emergency[cite: 46].

Documented Crisis

  • 3,012 kidnappings documented between Jan 2024 and April 2025[cite: 47].
  • Southwest states, particularly Oyo, are increasingly under threat[cite: 48, 49].

The Local Impact

  • Tragic abductions in Ogbomoso's Abaa area[cite: 50].
  • Estimated N2.23 trillion extracted in ransoms annually[cite: 51].

Why Traditional Systems Fail

The fundamental problem is the critical lack of a rapid, reliable, and accessible digital mechanism for alerting responders during crises[cite: 62].

Communication Gaps [cite: 64]

Traditional phone calls are often impossible during active threats where a victim cannot speak or is in an area with unstable cellular signals.

The Information Gap [cite: 66]

Responders frequently rely on vague verbal descriptions, leading to uncoordinated rescue efforts and high error rates[cite: 67, 126].

The Resilience Gap [cite: 68]

Standard web tools fail when connectivity is lost. In Ogbomoso transit routes, the inability to cache and sync data can lead to total loss of life-saving information[cite: 69].

Methodology: OOAD Framework

This study adopts the Object-Oriented Analysis and Design (OOAD) methodology. By modeling the ecosystem as interacting objects, we ensure each distress signal is a unique, trackable entity.

Core Theoretical Framework [cite: 104]

  • Event-Driven Architecture (EDA): Treats state changes as events, pushing location updates with millisecond latency[cite: 105].
  • Offline-First Theory: Using Service Workers to intercept requests and cache content for unstable networks[cite: 106].
  • Human-Centered Design: Focuses on "Panic-Proof" models to reduce cognitive load.

Technical Implementation

Frontend (PWA) [cite: 88]

Built with Next.js and TailwindCSS. We chose PWA architecture to solve the "Technological Inaccessibility" problem—no app store required, low data usage, and offline resilience[cite: 58, 59].

Backend (Real-Time) [cite: 89]

Utilizing Node.js and Socket.io for bi-directional distress signals. Socket.io ensures signal transmission even in restrictive network environments[cite: 117].

Persistence & Data [cite: 90]

PostgreSQL for reliable geospatial logging and forensic incident history[cite: 177, 190].

SafeBand Implementation

Victim Features [cite: 132]

  • One-Touch SOS: Instant activation for high-stress scenarios[cite: 133].
  • Live Geolocation: Continuous tracking stream to the server[cite: 134].
  • Background Sync: Auto-queues alerts if signal drops[cite: 181].

Responder Dashboard [cite: 135]

  • Map Visualization: Live mapping of active threats using Geolocation API[cite: 77].
  • Access Control: Authorized-only access to sensitive tracking data[cite: 189].

Conclusion & Impact

SafeBand represents a practical and cost-effective safety solution aimed at reducing response times and improving situational awareness[cite: 34].

Primary Significance [cite: 79]

  • Individual Level: Discreet and rapid alerting during narrow intervention windows[cite: 80].
  • Community Level: Tactical blueprint for data-driven policing for agencies like Amotekun[cite: 82, 83].
  • Academic Level: Advances LAUTECH research from GSM hardware to modern web ecosystems[cite: 84, 114].
